# descriptor-attestation

Reproduction code and cached results for the journal paper *Descriptor attestation for federated quantum calibration* (submitted to *Quantum Science and Technology*, 2026).

## Summary

This repository implements an attestation protocol that detects misreported hardware noise descriptors in federated quantum-error-mitigation settings. The protocol runs a fixed ensemble of Clifford witness circuits against each participating device and uses a deployment-prior-trained linear predictor to flag claimed descriptors whose maximum per-witness deviation exceeds a per-device threshold ε_device = ε_Hoeffding + ε_pred(d) + ε_model(d). Empirical results on three IBM Heron r2 backends (Kingston, Fez, Marrakesh) characterise completeness, attack detection across four attack families, federation scaling, deployment calibration, and three baseline comparisons (per-device-ε, Hoeffding-only, Stricker-adapted L1-mean).

**QPU access is not required** to reproduce the manuscript tables and figures. All hardware measurements used in the paper are cached as `data/raw/hardware/<backend>/run.npz`. Fresh IBM Quantum runs can be reproduced via `scripts/optional_ibm_runs/` but are optional.

## Installation

```bash
git clone https://github.com/CHYMEE/descriptor-attestation.git
cd descriptor-attestation
conda env create -f environment.yml
conda activate descriptor-attestation
```

Or with pip:
```bash
pip install -r requirements.txt
```

Python 3.10+ required. The only quantum dependency is `qiskit-aer` (density-matrix simulator). No IBM Quantum credentials needed.

## Quick start — reproduce everything

```bash
bash scripts/reproduce_all.sh
```

This runs scripts 01..08 in sequence (~5-10 minutes total wall, dominated by ~70s of predictor training in script 02). Outputs land in `tables/generated/` (Markdown + CSV) and `figures/final/` (PDF + PNG).

## Data description

See `docs/data_dictionary.md` for column-by-column descriptions of every data file. Brief summary:

- **Hardware baselines**: each `data/raw/hardware/<backend>/run.npz` contains the 50-witness sample-mean vector measured on IBM Heron r2 hardware at 1024 shots, plus the calibration-time noise descriptor (T1, T2, gate-error, readout-error for 4 qubits).
- **Calibration analysis**: `data/raw/calibration/per_device_epsilon_demo.json` contains the per-device ε_pred / ε_model / ε_device decomposition (Section 5.B).
- **Attack records**: `data/raw/attacks/attack_sweep_records.json` contains all 170 attack records (85 phase-1 + 85 phase-2) with claimed descriptors, observed max-deviations, and per-baseline flag outcomes.
- **Processed**: `data/processed/section*` holds downstream analysis JSONs (CIs, McNemar, replication margins, threshold characterization).
